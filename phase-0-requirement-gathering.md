# Phase 0 - Requirement Gatherings

- Objective: Teach requirements engineering through stakeholder analysis and user story definition.
- Timeframe: February
- Client: C. Kima S.J. Wea

## Key Components
- Interview scripts with stakeholder
- User story cards
- Functional and non-functional requirement templates

## Activities
- Conduct a mock stakeholder interview.
- Develop at least 6 user stories for the portfolio site.
- Create requirements document with clearly identified scope.

## Lesson Plan
- Week 1: Stakeholder interviews and requirements fleshing.
- Week 2: User story creation and acceptance criteria.
- Week 3: Requirement documents and scope definition.

## Assessment
- Peer validation of the requirement document.
- Instructor review of user stories for completeness and clarity.
